# Claude Code 入门教程 — 设计与内容建议

> 如何为「完全零基础的计算机小白」制作一篇看得懂、学得会、用得到的 Claude Code HTML 教程。

---

## 一、受众画像与教学策略

### 受众特点
- 从未接触过编程，不知道「终端」「命令行」「npm」是什么
- 会基本电脑操作（打开浏览器、下载文件、双击安装）
- 可能对「敲代码」有畏惧心理

### 教学策略
| 策略 | 说明 |
|------|------|
| 类比优先 | 每个陌生概念先用生活类比解释，再给技术定义 |
| 先做后懂 | 不要让读者先「学理论」，而是先跟着操作，看到成果后再解释原理 |
| 截图全覆盖 | 每一步操作都要有截图，读者照猫画虎即可 |
| 容错设计 | 预先列出常见错误和解决方法，降低挫败感 |
| 成就感驱动 | 每章结束有一个「能看到的结果」（生成了一个网页、运行了一段代码等） |

## 二、教程章节规划（共 8 章）

### 第 0 章：开始之前 — 三个你必须知道的概念

**目的**：消除陌生感，建立基本认知

- **什么是「终端/命令行」？**
  - 类比：你平时用鼠标点来点去操作电脑 → 这是「图形界面」；终端就是「打字界面」，你打一行字，电脑执行一行
  - 实操：教读者打开终端（Windows: Win+R → 输入 powershell → 回车；Mac: 聚焦搜索 → 输入 terminal → 回车）
  
- **什么是 Node.js？**
  - 类比：手机能运行 App 是因为有 iOS/Android 系统；Node.js 让你的电脑能运行 JavaScript 写的程序。Claude Code 就是一个 JavaScript 程序，所以需要 Node.js
  
- **什么是 Claude Code？**
  - 一句话：在你的终端里运行的 AI 程序员助手，你说需求，它帮你写代码、修 bug、部署项目

### 第 1 章：安装 Node.js

**目的**：搭建基础环境

- Windows 安装步骤（图文详细 + 截图占位符）
  1. 打开浏览器，访问 https://nodejs.org
  2. 点击左侧 LTS 版本下载
  3. 双击下载的 .msi 文件，一路点 Next
  4. 验证：打开 PowerShell，输入 `node -v`（显示版本号 = 成功）
  
- Mac 安装步骤
  1. 同上官网下载 .pkg
  2. 或使用 Homebrew：`brew install node`
  3. 验证同 Windows

- 常见坑：安装完要关掉终端重新打开

### 第 2 章：安装 Claude Code

**目的**：核心安装步骤

- 打开终端
- 输入命令：`npm install -g @anthropic-ai/claude-code`
  - 解释：`npm install` = 从应用商店下载，`-g` = 全局安装（所有地方都能用），`@anthropic-ai/claude-code` = 包名
- 等待下载完成（约 1-3 分钟）
- 验证：`claude --version`

### 第 3 章：注册与认证

**目的**：让 Claude Code 能用

- 两种认证方式对比：
  | 方式 | 难度 | 适合谁 |
  |------|------|--------|
  | 浏览器登录 | ⭐ 简单 | 新手首选 |
  | API Key | ⭐⭐ 中等 | 有开发经验 |

- 浏览器登录步骤：
  1. 终端输入 `claude`，回车
  2. 选择浏览器登录
  3. 自动弹出浏览器页面
  4. 注册/登录 Anthropic 账号
  5. 回到终端，显示认证成功

- API Key 方式（备选）：
  1. 访问 https://console.anthropic.com
  2. 注册并创建 API Key
  3. 终端执行设置命令

### 第 4 章：第一次对话

**目的**：让读者立刻看到效果

- 启动：`claude`
- 第一个问题建议："帮我创建一个显示'你好，世界！这是我用 AI 做的第一个网页'的 HTML 文件"
- 发生了什么（中文解释）：
  1. Claude Code 理解你的需求
  2. 自动生成代码
  3. 自动创建文件
  4. 你在文件夹里双击就能看到网页
- 常用操作清单（发送消息 / 退出 / 查看文件）

### 第 5 章：三个实战小项目

**目的**：巩固学习，展示实际价值

#### 项目 1：制作个人名片网页
- 指令："帮我做一个个人名片网页，包含我的名字、职业、联系方式，好看一点的"
- 学到的技能：描述需求 → AI 生成 → 预览 → 提修改意见

#### 项目 2：让 AI 帮你理解代码
- 场景：网上复制了一段代码看不懂
- 指令："帮我解释一下这段代码每一行在做什么：[粘贴代码]"
- 学到的技能：代码阅读理解

#### 项目 3：修改现有网页
- 指令："帮我把刚才的名片网页背景改成渐变色，字体变大一点"
- 学到的技能：迭代修改

### 第 6 章：常见问题与排错

**目的**：减少挫败感，独立解决问题

- ❌ "npm 不是内部或外部命令" → Node.js 没装好或环境变量问题
- ❌ 安装报错 EACCES → 权限不足，加 `sudo`（Mac/Linux）或以管理员运行（Windows）
- ❌ 安装速度慢 → 换淘宝镜像：`npm config set registry https://registry.npmmirror.com`
- ❌ Claude Code 启动报错 → 检查 Node.js 版本是否 ≥ 18
- ❌ 网络连接失败 → 检查代理/VPN 设置

### 第 7 章：下一步学什么

**目的**：引导持续学习

- 官方文档：https://docs.anthropic.com/en/docs/claude-code
- 进阶功能预览：项目模式、/commands 快捷指令、MCP 工具扩展
- Claude Code 社区

## 三、HTML 设计规范（10 条）

### 1. 单文件、零依赖
```css
/* 所有样式内联在 <style> 标签，无外部 CSS/JS 引用 */
/* 双击 .html 文件即可在浏览器打开 */
```

### 2. 响应式布局
- 桌面：内容区最大宽度 800px，居中，左侧固定目录
- 平板/手机：目录折叠为顶部汉堡菜单，内容区占满宽

### 3. 配色方案
- 主背景：#FAFBFC（极浅灰白）
- 主文字：#1A1A2E（深蓝黑，不刺眼）
- 强调色：#4F46E5（靛蓝，按钮/链接）
- 代码块背景：#1E1E2E（深色）
- 提示框：绿（贴士）/ 黄（注意）/ 红（错误）

### 4. 排版
- 正文字体大小：16px
- 标题从 32px 递减
- 行高 1.8
- 段落间距 1.5em
- 代码块字体：'Fira Code', 'Consolas', monospace

### 5. 代码块
- 深色背景 + 语法高亮（简单版）
- 右上角**复制按钮**（点击复制整段命令）
- 每行命令配 `# ← 这行命令在做什么` 的中文注释

### 6. 截图占位符
```css
.screenshot-placeholder {
  border: 2px dashed #CBD5E1;
  border-radius: 12px;
  background: #F1F5F9;
  padding: 40px 20px;
  text-align: center;
  color: #64748B;
}
```
- 每个占位符内写明：「📸 此处放一张截图：[具体描述截图内容]」

### 7. 导航系统
- 桌面端：左侧 240px 固定侧边栏，章节链接 + 当前高亮
- 移动端：顶部固定导航栏 + 汉堡菜单

### 8. 步骤编号
- 大号圆形数字（40px 直径），靛蓝色背景白色数字
- 每步配 emoji 图标

### 9. 提示框 3 种
```html
<div class="tip">     💡 小贴士：...</div>
<div class="warning">  ⚠️ 注意：...</div>
<div class="error">   ❌ 常见错误：...</div>
```

### 10. 进度条
- 页面顶部固定 3px 高度进度条
- 随滚动实时更新百分比
- 纯 CSS + 少量 JS（scroll 事件监听）

## 四、写作注意事项

1. **避免术语轰炸**：每个新技术名词出现时，必须用括号给出大白话解释
2. **人称亲和**：使用「你」而非「用户」，营造一对一教学的氛围
3. **鼓励性语言**：遇到等待步骤时说「喝口水，等它装完」，安装成功时说「🎉 恭喜！你已经完成最难的一步了」
4. **命令必须可复制**：代码块内的命令不要有多余字符，确保复制粘贴即可执行
5. **讲「为什么」**：不只告诉读者敲什么命令，还要用 1-2 句话解释这个命令在干什么
