# Claude Code 入门教程 — AI 提示词模板

> 两个版本：**简短版**适合快速使用，**完整版**适合面试展示 prompt 工程能力。

---

# 一、简短版（~15 行，复制即用）

---

```
你是一位资深技术文档专家。请用纯中文生成一份适合「完全零基础计算机小白」的
Claude Code 安装与使用教程，输出为单个 .html 文件。

要求：
1. 从最基础的概念讲起（什么是终端、Node.js、npm），每个概念用生活类比
2. 包含 8 章：基础概念→安装Node.js→安装Claude Code→注册认证→第一次对话→3个实战项目→常见问题排错→进阶资源
3. 所有命令代码配中文注释，代码块有复制按钮
4. 截图位置用虚线占位符 + 说明文字（不用真实图片）
5. 响应式单文件 HTML，浅色主题，16px+ 大字体，3 种提示框（小贴士/注意/错误）
6. 语气亲和、鼓励性，像一位耐心的老师在对学生说话
7. 直接输出完整的 HTML 代码，不要废话
```

# 二、完整版（面试展示用，~80 行）

---

```
# 角色设定
你是一位拥有 10 年经验的技术文档工程师，专长是将复杂的技术概念转化为
零基础用户也能看懂的教程。你的写作风格是：亲和、耐心、循序渐进。
你会用生活中的比喻来解释技术概念，会在每一步操作后给读者正向鼓励。

# 任务目标
生成一份 Claude Code 安装与使用的入门教程，输出为单个 .html 文件。
读者只需双击文件就能在浏览器中打开、阅读、跟着操作。

# 目标受众
- 完全零基础的计算机小白
- 从未接触过编程，不知道「终端」「命令行」「Node.js」是什么
- 会基本的电脑操作（打开浏览器、下载文件、双击安装程序）
- 对「敲代码」有一定的畏惧心理，需要被鼓励

# 内容结构（必须严格包含以下 8 章）

## 第 0 章：开始之前 — 三个你必须知道的概念
- 什么是终端/命令行（类比：图形界面 vs 打字界面）
- 什么是 Node.js / npm（类比：手机系统 vs 应用商店）
- 什么是 Claude Code（一句话：你的 AI 编程助手）
- 教读者如何打开终端（Windows: Win+R→powershell, Mac: 搜索→terminal）

## 第 1 章：安装 Node.js
- Windows 和 Mac 分别说明
- 官网下载、双击安装、验证「node -v」「npm -v」
- 常见坑：装完要关掉终端重新打开

## 第 2 章：安装 Claude Code
- 命令：npm install -g @anthropic-ai/claude-code
- 解释这条命令每部分在做什么
- 验证：claude --version

## 第 3 章：注册与认证
- 浏览器登录方式（推荐新手）+ 详细步骤
- API Key 方式（备选）+ 步骤
- 两种方式的对比表格

## 第 4 章：第一次对话
- 启动：claude
- 第一个问题建议：「帮我创建一个写有"你好世界"的网页」
- 解释 Claude Code 在背后做了什么
- 常用操作清单

## 第 5 章：三个实战小项目
- 项目 1：制作个人名片网页
- 项目 2：让 AI 帮你解释看不懂的代码
- 项目 3：修改网页的颜色和布局
- 每个项目包含：需求指令 + 预期结果 + 学到了什么

## 第 6 章：常见问题与排错
- "npm 不是内部或外部命令"
- 安装报权限错误
- 安装速度慢的镜像切换方案
- 网络/代理连接问题
- 每个错误用 ❌ 开头，解决方案用 ✅ 开头

## 第 7 章：下一步学什么
- 官方文档链接
- 进阶功能简介
- 学习路径建议

# 语言要求
- 全文使用简体中文，命令和代码保留英文原文
- 每条命令后配中文注释（用 # 号），解释这条命令在做什么
- 每个技术名词首次出现时用括号给出大白话解释
- 称呼读者为「你」，营造一对一教学的亲近感
- 在读者完成困难步骤后给予鼓励（如「🎉 恭喜，你已经完成了最难的一步！」）

# HTML 设计规范（必须遵守）

1. **单文件自包含**：所有 CSS/JS 写在 <style> 和 <script> 标签内，无外部依赖
2. **响应式布局**：桌面端内容区 800px 居中 + 左侧 240px 固定导航；
   移动端导航折叠为顶部汉堡菜单
3. **配色**：浅色主题，柔和蓝白配色。背景 #FAFBFC，主文字 #1A1A2E，
   强调色 #4F46E5，代码块深色背景 #1E1E2E
4. **排版**：正文字体 16px 以上，行高 1.8，段落间距宽松
5. **代码块**：深色背景，右上角有「📋 复制」按钮，点击复制整段代码
6. **截图占位符**：虚线边框区域 + emoji + 具体说明文字，样式如下：
   - border: 2px dashed #CBD5E1; border-radius: 12px;
   - background: #F1F5F9; padding: 40px; text-align: center;
   - 文字格式：「📸 此处放一张截图：[具体描述]」
7. **目录导航**：桌面端固定在左侧，当前阅读章节高亮
8. **步骤编号**：每个操作步骤用大号圆形数字（40px 靛蓝色圆底白字）
9. **提示框 3 种类型**：
   - 💡 小贴士（绿色左边框）
   - ⚠️ 注意（黄色左边框）
   - ❌ 常见错误（红色左边框）
10. **进度条**：页面顶部固定 3px 高的阅读进度条，随滚动实时更新
11. **平滑滚动**：点击目录项平滑滚动到对应章节，使用 scroll-behavior: smooth

# 截图策略
- 所有需要截图的位置使用 CSS 虚线占位符
- 占位符内标明「📸 + 具体截图内容描述」
- 不需要任何真实的图片文件或 base64 编码

# 输出格式
- 直接输出完整的 HTML 代码，不要任何额外说明
- 代码开头是 <!DOCTYPE html>，结尾是 </html>
- 所有内容写在一个文件中

# 质量要求
- 可读性：一个 60 岁的老人家也能看懂
- 趣味性：避免枯燥的技术文档口吻，多用比喻和故事
- 实操性：读者看完后能真正完成安装并成功使用 Claude Code
- 完整性：不遗漏任何前置步骤，不假设读者已经知道任何技术知识
- 美观性：HTML 页面干净、专业、现代，像一个正式的产品文档页面
```

# 三、使用说明

## 如何使用简短版
1. 复制简短版提示词
2. 粘贴到 Claude、ChatGPT 或任何 AI 对话工具
3. 等待 AI 生成完整 HTML
4. 将输出内容保存为 `claude-code-guide.html`
5. 双击打开，检查效果
6. 根据需要对某些章节进行调整

## 如何使用完整版
1. 完整版更适合在面试场景展示你的 prompt 工程能力
2. 向面试官说明：「这是我为这个任务设计的提示词，包含角色设定、受众分析、结构约束、设计规范和质量标准」
3. 可以现场演示：将提示词粘贴到 AI 工具，展示生成效果
4. 生成后，可以补充真实的截图替换占位符

## 后续优化建议
- 替换截图占位符为真实操作截图
- 根据反馈调整语言难度
- 补充更多平台（Linux）的安装说明
- 添加评论区或反馈表单
- 增加暗色模式切换
